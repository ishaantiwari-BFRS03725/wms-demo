# SPA Migration Design

**Date:** 2026-06-19
**Goal:** Remove the Node.js server requirement by migrating from TanStack Start (SSR) to plain TanStack Router (SPA). Deploy static files via nginx in Docker.

---

## 1. Dependencies

**Remove:**
- `@tanstack/react-start` (runtime)
- `@lovable.dev/vite-tanstack-config` (devDep — wraps Nitro + Start)
- `nitro` (devDep)

**No additions needed** — `@vitejs/plugin-react` and `@tanstack/router-plugin` are already in devDeps.

---

## 2. Files to Delete

| File | Reason |
|------|--------|
| `src/server.ts` | SSR server entry — wraps `@tanstack/react-start/server-entry` |
| `src/start.ts` | TanStack Start middleware entry |
| `src/lib/error-page.ts` | Server-side HTML error page renderer |
| `src/lib/error-capture.ts` | Server-side error capture for h3/Nitro |
| `src/lib/config.server.ts` | Server-only env config |
| `src/lib/api/example.functions.ts` | Example `createServerFn` (not used in app) |

## 3. Files to Create

### `index.html` (project root)
Standard Vite SPA entry — references `src/main.tsx`.

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Shiprocket WMS</title>
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.tsx"></script>
  </body>
</html>
```

### `src/main.tsx`
Client entry — creates React root and renders the router.

```tsx
import { StrictMode } from "react";
import ReactDOM from "react-dom/client";
import { RouterProvider } from "@tanstack/react-router";
import { getRouter } from "./router";

const router = getRouter();

ReactDOM.createRoot(document.getElementById("root")!).render(
  <StrictMode>
    <RouterProvider router={router} />
  </StrictMode>
);
```

## 4. Files to Modify

### `vite.config.ts`
Replace `@lovable.dev/vite-tanstack-config` with standard Vite + TanStack Router plugin.

```ts
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import { TanStackRouterVite } from "@tanstack/router-plugin/vite";
import tailwindcss from "@tailwindcss/vite";
import tsconfigPaths from "vite-tsconfig-paths";

export default defineConfig({
  plugins: [TanStackRouterVite(), react(), tailwindcss(), tsconfigPaths()],
  server: { allowedHosts: true },
});
```

### `src/routes/__root.tsx`
Remove SSR-only APIs: `shellComponent`, `HeadContent`, `Scripts`, `head()`. The root component becomes a plain React component. Meta tags move to `index.html`. Google Fonts link moves to `index.html`. Error and NotFound components stay unchanged.

### `package.json` scripts
Update `build:server` → `build` (or keep `build` as-is since `npm run build` already calls `vite build`). Remove any Start-specific scripts if present.

## 5. Docker / nginx

### `Dockerfile`
Replace the Node.js runner stage with `nginx:alpine`. Build stage stays identical (runs `npm run build` → outputs to `dist/`).

```dockerfile
FROM node:22-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
  CMD wget -qO- http://localhost/ || exit 1
CMD ["nginx", "-g", "daemon off;"]
```

### `nginx.conf` (new file at project root)
SPA fallback: all unknown paths serve `index.html` so client-side routing works.

```nginx
server {
  listen 80;
  root /usr/share/nginx/html;
  index index.html;

  location / {
    try_files $uri $uri/ /index.html;
  }

  gzip on;
  gzip_types text/css application/javascript application/json image/svg+xml;
}
```

### `docker-compose.yml`
Update port mapping from `3000:3000` → `3000:80` (nginx listens on 80 inside container). Update healthcheck from `localhost:3000` → `localhost:80` (or just `localhost/`).

### `k8s/deployment.yaml`
Update `containerPort` from `3000` → `80`. Health probe paths (`/`) stay unchanged.

---

## 6. What Does NOT Change

- All 70+ route files in `src/routes/` — untouched
- All mock data in `src/lib/wms/` — untouched
- All components in `src/components/` — untouched
- `src/router.tsx` — untouched
- `src/routeTree.gen.ts` — auto-regenerated by TanStack Router plugin on build
- Route `loader` functions — work identically in SPA mode (run client-side)
- `src/lib/lovable-error-reporting.ts` — keep (browser-safe)

## 7. Build Output

| Before | After |
|--------|-------|
| `.output/server/index.mjs` (Node server) | `dist/` (static HTML/JS/CSS) |
| `node .output/server/index.mjs` | `nginx` serving `dist/` |
| 256MB memory limit needed | ~10MB nginx container |
